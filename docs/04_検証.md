# 04. 検証

Cursor で本リポジトリを開いた後の **動作確認手順**。各ステップを上から叩いて、想定どおりに動いているかを判定する。

すべての操作は **Windows 上の Cursor アプリ** で実行する。

---

## 1. 検証ゲートの考え方

| ゲート | 内容 | 失敗時の典型対処 |
|---|---|---|
| G1 | Cursor を起動できる / サインイン済み | 再インストール、Cursor のサインインやり直し |
| G2 | リポジトリを Open Folder で開けて、ファイルツリーが見える | Open Folder のパス再確認、フォルダ信頼の許可 |
| G3 | Chat (Ctrl+L) を開いてプロンプトに応答が返る | Hobby 枠の上限到達確認、モデル選択確認、サインイン状態確認 |

---

## 2. Cursor 起動とサインイン確認

### G1: 起動とバージョン

1. Cursor を起動
2. メニュー: `Help → About` → バージョン情報が表示されれば G1 通過
3. 右下のステータスバーに自分のアカウント名が表示されているか確認（未サインインなら `Sign in` ボタンが出る）

---

## 3. リポジトリを開けることの確認

### G2: Open Folder

1. `File → Open Folder` → `study-cursor/` を選択
2. 初回は「このフォルダを信頼しますか？」と聞かれるので `Yes, I trust the authors`
3. 左サイドバーに以下が表示されれば G2 通過
   - `README.md`
   - `AGENTS.md`
   - `.cursor/` フォルダ
   - `docs/` フォルダ
   - `src/office-task/` フォルダ

### `.cursor/rules/repo.mdc` の読み込み確認

1. Chat (Ctrl+L) を開く
2. 入力欄上部または右ペインに「Rules」「Applied rules」のような表示があり、`repo.mdc` がアクティブになっていれば、プロジェクトルールが Chat のコンテキストに加わっている

---

## 4. プロンプト軽量実行の確認

### G3: Chat に最小プロンプトを投げる

1. Chat (Ctrl+L) を開く
2. 以下を貼って送信（**プロンプト4: エンジニア一覧 / 案件一覧の集計レポート** を最小形で実行。ファイル名は曖昧でも `@file` で読めば中身は判別できる）:

   ```
   @file src/office-task/data/新規 データ1.csv の内容を読み、以下を Markdown で出力してください。
   - このファイルの中身が何のデータかを 1 行で要約
   - レコード件数
   - 主力スキル（primary_skill）別の人数
   - 月単金（monthly_rate）の平均と中央値
   ```

3. Cursor が以下を返せば G3 通過:
   - 「自社・協力会社の稼働中エンジニア一覧」のような中身の要約
   - 件数（10 件）、主力スキル別の集計、単金の平均・中央値を含む Markdown 表
   - Python の `pandas` などを使わず、AI が CSV を直接読んで集計した解

### よくある問題

| 症状 | 原因 / 対処 |
|---|---|
| Chat の応答に「Python の `os` モジュールで…」など Python コードが含まれる | `AGENTS.md` と `.cursor/rules/repo.mdc` が読まれていない。Open Folder を一度閉じて開き直す。または Chat に「ランタイム非インストール原則に従い PowerShell / Excel / Cursor 内処理で完結する解を再提示してください」と追記して再依頼 |
| Chat の応答が極端に短い / モデルが落ちる | Hobby 枠の slow request 上限に近い可能性。詳細は [`./教育資料/04_Cursor操作.md`](./教育資料/04_Cursor操作.md) |
| `@file` でファイル添付しても中身を読まない | Cursor のインデックス未完了。少し待ってから再試行 |

---

## 5. 一括チェックリスト

クリーンな Windows に Cursor をインストールした直後に、以下を上から実施して全部 OK になれば最低限の動作確認は完了。

```
[ ] Cursor アプリが起動する
[ ] Help → About でバージョンが表示される
[ ] サインイン済み（右下にアカウント名が出る）
[ ] File → Open Folder で study-cursor/ を開ける
[ ] 左サイドバーに README.md / AGENTS.md / src/office-task/ が見える
[ ] Chat (Ctrl+L) が開ける
[ ] §4 の最小プロンプトに対し、Python を使わない解（CSV を直接読んだ集計結果）が返る
```

すべてチェックがついたら、`docs/05_運用.md §3` の日常運用に進む。
