# 05. 運用

利用者が **このリポジトリを使って Cursor で実作業を始めるまで** の手順と、日常的な使い回しの流れ。

---

## 1. 前提

- **対象 OS は Windows 10/11 のみ**
- **ランタイムはインストールしない**（Python / Node.js なし）
- 以下の手順は **Cursor の Chat / Composer**（Windows 上のアプリ）で実行する想定

---

## 2. 初回セットアップ（Cursor インストール → リポジトリを開く）

### STEP 0. Cursor をインストールしてサインインする

1. [cursor.com](https://cursor.com/) にアクセス
2. 「Download for Windows」から `.exe` インストーラを取得して実行
3. インストール完了後に Cursor を起動 → サインイン（無料アカウント = Hobby 枠で OK）

詳細は [`./教育資料/01_環境構築ガイド.md`](./教育資料/01_環境構築ガイド.md) を参照。

### STEP 1. リポジトリを取得する

**方法 A（推奨・最短）**: Cursor の Welcome 画面 → `Clone repo` から URL を貼って取得

**方法 B**: Windows 標準の `git`（無ければ [git-scm.com](https://git-scm.com/) からインストール）または PowerShell で `git clone`

```powershell
# Windows の cmd / PowerShell で実行
mkdir C:\dev
cd C:\dev
git clone https://github.com/kurosawa-kuro/study-cursor.git
```

> 💡 配置場所はどこでも構わない（`C:\dev\` / `%USERPROFILE%\Documents\` など）。

### STEP 2. Cursor で Open Folder

1. Cursor を起動
2. `File → Open Folder` で取得した `study-cursor/` フォルダを選択
3. 初回は「このフォルダを信頼するか？」と聞かれるので Yes
4. 左サイドバーに `README.md` `AGENTS.md` `docs/` `src/` が見えれば OK

### STEP 3. 動作確認

[`04_検証.md`](./04_検証.md) の手順を上から実行して全部緑になれば最低限 OK。

---

## 3. 日常の作業フロー

### 3.1 事務処理タスク（`src/office-task/`）

1. Cursor の左サイドバーから [`src/office-task/README.md`](../src/office-task/README.md) を開く
2. 使いたいプロンプトセクション（例: `## プロンプト1: 商談後のお礼・提案メール文面生成`）をまるごとコピー
3. **Chat を開く**: `Ctrl+L`（または右上の Chat アイコン）
4. コピーしたプロンプトを貼り付け、必要に応じて自分の業務情報（ファイル名・宛先・要件等）を補って送信
5. Cursor が処理（`@file` でサンプルデータを読ませる／PowerShell ワンライナーを返してもらう／Excel 数式を提示してもらう など）
6. 結果のファイル操作は Cursor の編集 / PowerShell の手元実行 / Excel への手動コピペ で適用

> プロンプトは **共通要件を含めて自己完結している** ので、1 セクションそのままコピペで成立する設計。

#### 重要な約束

- **元ファイルは変更・移動・削除しない**。コピーして新しい名前を付けて保存させる
- 結果が想定外なら、ロールバックは「コピー側を消すだけ」で済む
- ランタイム導入を伴う解（`pip install ...` 等）が AI から返ってきたら、**Windows 標準機能（PowerShell / Excel）または Cursor 内処理で代替できないか** を AI に再依頼する

### 3.2 Cursor の機能の使い分け

| 操作 | キー | 用途 |
|---|---|---|
| Chat | `Ctrl+L` | 質問・調査・小さなコード生成・テキスト整形 |
| Composer | `Ctrl+I` | 複数ファイル横断の変更（Hobby 枠の上限に注意） |
| Inline Edit | `Ctrl+K` | エディタ内のピンポイント編集 |
| Tab 補完 | `Tab` | 入力中の補完候補を採用 |

詳細・`@file` / `@folder` / `@docs` の使い方は [`./教育資料/03_Cursor操作.md`](./教育資料/03_Cursor操作.md) を参照。

## 4. メンテナンス

### 4.1 Cursor のアップデート

Cursor は自動的に更新される。手動更新は `Help → Check for Updates`。

### 4.2 リポジトリ自体の更新

Cursor の Source Control パネル（左サイドバーの `Ctrl+Shift+G`）から `Pull`、または PowerShell で:

```powershell
cd C:\dev\study-cursor
git pull
```

プロンプトメモやドキュメントの追加・修正が反映される。

### 4.3 ログイン期限切れ等

| 症状 | 対処 |
|---|---|
| Cursor 起動時にサインイン画面が出る | 画面の指示に従ってサインインし直す（Hobby 枠の無料アカウントで OK） |
| Chat が `Too many slow requests` 等で止まる | Hobby 枠の月次上限に到達。詳細・上限値は [`./教育資料/03_Cursor操作.md`](./教育資料/03_Cursor操作.md) を参照 |
| Chat の応答が極端に短い・モデルが切り替わった | Hobby 枠で利用可能なモデルに自動フォールバックされている可能性。モデル選択を見直す |
