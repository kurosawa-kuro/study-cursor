## このガイドの目的

**IT エンジニア派遣（SES）の営業担当者** が、**Cursor（Hobby 枠 = 無料プラン）を活用して営業事務を自動化できるようにする** ための環境構築ガイドです。

| 業務シーン | 自動化できる例 |
| --- | --- |
| 顧客対応 | 商談後のお礼・提案メール文面生成、商談議事録テンプレ生成 |
| エンジニア・案件管理 | エンジニア一覧 / 案件一覧の集計レポート、エンジニア × 案件マッチング（CSV 結合） |
| 提案資料作成 | スキルシート・契約書の構造化・要約 |
| ベンダー比較 | 協力会社見積書 PDF → 比較用 CSV（複数ベンダー） |
| ファイル整理 | ファイル名リネーム（スキルシート整理）、拡張子別フォルダにコピー |

### ゴール

- **事務処理** を Cursor に頼んで自動化できる（コードが書けなくても、日本語で指示すれば OK）
- **Cursor の Hobby 枠（無料）だけ** で完結する
- **ランタイム不要**（Python・Node.js のインストール無し）

### なぜ Cursor を使うのか

Cursor は **VSCode をベースに AI コーディング支援を全面統合した Windows ネイティブアプリ**。

| 比較項目 | Cursor | 一般的な AI チャット（ChatGPT 等のブラウザ版） |
|---|---|---|
| ファイル/フォルダ全体を読ませる | `@file` `@folder` で直接添付 | コピペで都度貼り付け |
| 編集結果をそのまま反映 | エディタ統合で直接書き戻し | コピペで手動反映 |
| プロジェクトルール | `AGENTS.md` / `.cursor/rules/*.mdc` | 毎回プロンプトで指定 |
| Hobby 枠 | 無料アカウントで月次の slow request 枠あり | 無料枠あり |

事務処理のように **特定のフォルダ / ファイル群を扱う作業** では、Cursor の `@file` / `@folder` 機能が圧倒的に効きます。

### Cursor Hobby 枠とは

> Cursor の **無料プラン**。月単位で **slow request** を一定数使える。具体的な数値（リクエスト数・利用可能モデル）は変動するため [`04_Cursor操作.md`](./04_Cursor操作.md) を参照。

本キットは **Hobby 枠の上限内で完結する設計**（プロンプトを 1 タスク 1 リクエストに収め、長文・多並列を避ける）。

## ① Cursor をダウンロード

### 手順

**1. ブラウザで Cursor 公式サイトを開く**
   - https://cursor.com/

**2. ダウンロードボタンをクリック**
   - 「Download for Windows」と書かれたボタンをクリック
   - インストーラ（`Cursor Setup-x.x.x.exe`）がダウンロードされます

> 💡 ファイルサイズはおおよそ 100〜200 MB 程度。ネットワーク環境により数十秒〜数分かかります。

---

## ② Cursor をインストール

### 手順

**1. ダウンロードした `.exe` を実行**
   - エクスプローラの「ダウンロード」フォルダで `Cursor Setup-x.x.x.exe` をダブルクリック
   - ユーザーアカウント制御（UAC）が出たら「はい」を選択

**2. インストーラの指示に従う**
   - ライセンス同意 → インストール先（既定のままで OK） → `Install`
   - 完了後、`Finish` をクリックすると Cursor が自動起動します（または手動でスタートメニューから `Cursor` を選択）

---

## ③ Cursor にサインイン（Hobby = 無料）

### 手順

**1. Cursor 起動直後の Welcome 画面で `Sign in` または `Sign up` をクリック**

**2. ブラウザが開いて Cursor のサインインページが表示される**
   - 既存アカウントがあればサインイン
   - 無ければ GitHub / Google / メールアドレスのいずれかで新規アカウントを作成
   - **新規アカウントはデフォルトで Hobby 枠（無料）** になります

**3. 認証完了後、ブラウザから Cursor 本体に戻る**
   - ブラウザに「Cursor を開きますか？」と聞かれたら許可
   - Cursor の右下にアカウント名が表示されればサインイン成功 ✅

### サインイン時の安心ポイント（Hobby 枠 = 無料）

非エンジニアの方が不安になりやすい点を、先に整理しておきます。

```
   ✅ クレジットカードは不要
      → Hobby 枠は Free。カード番号を一切登録せずに使い始められる
        （課金が発生するのは Pro / Teams を自分で選んだときだけ）

   📱 SMS（電話番号）認証は必要
      → アカウント作成時に携帯電話の SMS でコード認証を求められる
        （本人確認のため。番号は認証用で、課金とは無関係）

   🐙 GitHub アカウントは必須ではない
      → サインインは GitHub / Google / メールアドレスのいずれかで OK
        GitHub を持っていなくても、メールアドレスだけで始められる

   🔒 コードのデータ共有は「必須ではない」
      → Cursor には Privacy Mode（プライバシーモード）があり、
        有効にすると入力したコード/ファイルが学習・保存に使われない
        本キットは社外秘を含まないサンプルデータ前提だが、
        実業務データを扱うなら Privacy Mode を ON にして使うと安心
```

> 💡 まとめると **「カード登録なし・メールだけで・無料で」始められ、
> データ共有も自分でオフにできる**。導入のハードルは低く設計されています。
> プライバシーの詳細は公式 [Privacy / Security](https://cursor.com/privacy) を参照。

## ④ 基本設定（任意）

### 言語を日本語にしたい場合

1. Cursor の `View → Command Palette`（または `Ctrl+Shift+P`）
2. `Configure Display Language` を選択
3. `日本語 (ja)` を選択 → Cursor を再起動

### テーマ・フォント

- `File → Preferences → Settings`（または `Ctrl+,`）から好みに合わせて変更

### プライバシーモードを有効にしたい場合（実業務データを扱うなら推奨）

1. `File → Preferences → Cursor Settings`（または右上の歯車アイコン → Cursor Settings）
2. `Privacy` / `Privacy Mode` の項目を探して **ON** にする
3. ON の状態では、入力したコード/ファイルが Cursor 側で学習・保存に使われません

> 💡 設定 UI の名称は Cursor のバージョンで変わることがあります。見つからない場合は
> 設定検索に `privacy` と入力。最新仕様は [`04_Cursor操作.md`](./04_Cursor操作.md) と公式 docs を参照。

## トラブルシューティング（環境構築フェーズ）

| 症状 | 対処法 |
| --- | --- |
| `Cursor Setup-x.x.x.exe` を実行すると「Windows によって PC が保護されました」と出る | 「詳細情報」→「実行」をクリック。または .exe を右クリック → プロパティ →「ブロックの解除」にチェック → OK 後にもう一度実行 |
| インストールが途中で失敗する | ダウンロードが破損している可能性。`.exe` を削除して cursor.com から再ダウンロード |
| サインインのブラウザが開かない | Windows の既定ブラウザを Edge / Chrome に設定。または Cursor 画面に表示される URL を手動でブラウザに貼ってサインイン完了させる |
| サインイン後 Cursor 本体に戻らない | ブラウザの「Cursor を開きますか？」を「許可」。または一度 Cursor を再起動 |
| Hobby 枠の上限に達したと言われる | 月次の slow request 上限に到達。詳細・対処は [`04_Cursor操作.md`](./04_Cursor操作.md) を参照 |
| Chat の応答が極端に短い／モデルが切り替わる | Hobby 枠で利用可能なモデルに自動フォールバックされている可能性。モデル選択を見直す |

> 💡 リポジトリを開いた後（`Open Folder` 以降）のトラブルは **[`ハンズオン`](./03_ハンズオン.md)** ドキュメント末尾を参照してください。
> Cursor の操作詳細（Chat / Composer / Tab / `@file`）は **[`04_Cursor操作.md`](./04_Cursor操作.md)** にまとめています。
