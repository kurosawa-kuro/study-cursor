## ハンズオン手順

### 全体フロー

```
   👤 あなた
    │
    ▼
   ① リポジトリ取得（Clone repo / git clone / ZIP DL のいずれか）
    │
    ▼
   ② Cursor で File → Open Folder → study-cursor/ を選択
    │
    ▼
   ③ 「このフォルダを信頼しますか？」→ Yes
    │
    ▼
   ④ Ctrl+L で Chat を開く
    │
    ▼
   ⑤ src/office-task/README.md のプロンプト1 を貼って送信
    │
    ▼
   🎉 Cursor が新ファイル名案 + PowerShell ワンライナーを返す
```

### ① リポジトリを取得する

3 つの方法のうち、好きなものを選んでください。

#### 方法 A: Cursor の Clone repo（推奨・git 不要）

1. Cursor の Welcome 画面（または `File → New Window`）に `Clone repo` ボタンが出る
2. URL 入力欄に貼り付け:
   ```
   https://github.com/kurosawa-kuro/study-cursor.git
   ```
3. 保存先フォルダを選択（例: `C:\dev\` や `%USERPROFILE%\Documents\`）
4. 取得完了後、自動でそのフォルダが Open される

#### 方法 B: PowerShell で `git clone`（git がインストール済みなら最速）

```powershell
# Windows の cmd / PowerShell で実行（管理者でなくて OK）
mkdir C:\dev   # 既にあれば不要
cd C:\dev
git clone https://github.com/kurosawa-kuro/study-cursor.git
```

git が入っていない場合は [git-scm.com](https://git-scm.com/) から「Git for Windows」をインストール。

#### 方法 C: GitHub から ZIP ダウンロード（git すら使いたくない場合）

1. ブラウザで https://github.com/kurosawa-kuro/study-cursor を開く
2. 緑の `Code` ボタン → `Download ZIP`
3. ダウンロードした zip をエクスプローラで右クリック → `すべて展開`
4. 展開先のフォルダ（例: `C:\dev\study-cursor`）を覚えておく

> 💡 取得方法は何でも OK。最終的に `study-cursor/` フォルダがローカルにあれば次に進めます。

### ② Cursor で Open Folder

1. Cursor を起動
2. `File → Open Folder`（または Welcome 画面の `Open Folder`）
3. 先ほど取得した `study-cursor/` フォルダを選択 → `フォルダーの選択`

### ③ フォルダ信頼の確認

初回は「このフォルダを信頼しますか？」というダイアログが出ます。

```
   Do you trust the authors of the files in this folder?
   
   [Yes, I trust the authors]   [No, I don't trust the authors]
```

`Yes, I trust the authors` をクリック。
左サイドバーに以下が表示されれば OK:

- `README.md`
- `AGENTS.md`
- `.cursor/` フォルダ
- `docs/` フォルダ
- `src/office-task/` フォルダ

### ④ Chat (Ctrl+L) を開く

`Ctrl+L` を押す（または右上の Chat アイコンをクリック）。
右ペインに Chat 欄が表示され、`.cursor/rules/repo.mdc` と `AGENTS.md` のルールが自動でコンテキストに加わります（Chat 上部に「Rules applied」のような表示が出る場合あり）。

### ⑤ プロンプト1（商談後のお礼・提案メール文面生成）を貼って動作確認

1. Cursor の左サイドバーから [`src/office-task/README.md`](../../src/office-task/README.md) を開く
2. `## プロンプト1: 商談後のお礼・提案メール文面生成` のセクション全体をコピー
3. Chat に貼り付け、最低限の入力情報（宛先・件名・要件メモ）を補って送信

期待される応答:

```
   ・件名 / 宛名 / 時候の挨拶 / 本文 / 結び / 署名欄を含む日本語ビジネスメール
   ・過剰な敬語にならない自然な表現
   ・必要に応じて 1〜2 案提示
```

慣れてきたら、次は以下を試してください（プロンプト4: エンジニア一覧 / 案件一覧の集計レポート）。`@file` 機能でサンプル CSV を直接読ませる動作を確認できます（ファイル名は意図的に曖昧で、Cursor に中身を判別させる演習も兼ねます）:

```
@file src/office-task/data/新規 データ1.csv の内容を読み、まずこのファイルが何のデータかを 1 行で要約し、続いて主力スキル別の人数と月単金の平均を Markdown 表で出してください。
```

応答が **Python コードや `pip install ...` の指示** を含む場合は、Chat に追記して再依頼:

```
ランタイム非インストール原則に従い、Python を使わず PowerShell / Excel / Cursor 内処理で完結する解を再提示してください。
```

## トラブルシューティング（ハンズオンフェーズ）

| 症状 | 対処法 |
| --- | --- |
| Cursor の `Clone repo` で `Authentication failed` | パブリックリポジトリなら認証不要。URL が `https://github.com/kurosawa-kuro/study-cursor.git` 形式か確認。プライベートなら配布元から HTTPS + Personal Access Token を取得 |
| `git: command not found`（PowerShell） | Windows に git が未インストール。[git-scm.com](https://git-scm.com/) から Git for Windows を入れる。または Cursor の `Clone repo` を使う |
| Open Folder してもファイルツリーが空 | 取得先のフォルダ階層を間違えている。展開後の `study-cursor/` フォルダを直接開くこと（その親フォルダではなく） |
| 「このフォルダを信頼しますか？」を `No` にしてしまった | `File → Manage Workspace Trust → Trust this folder` で許可し直す |
| Chat (Ctrl+L) が開かない | キーバインドが他のアプリと競合している可能性。右上の Chat アイコンをクリック、または `View → Chat` |
| Chat の応答に Python コードが含まれる | `AGENTS.md` と `.cursor/rules/repo.mdc` が読まれていない。Open Folder を一度閉じて開き直す。または「ランタイム非インストール原則に従い…」と追記して再依頼 |
| Chat が応答を返さない／途中で止まる | Hobby 枠の slow request 上限に近い可能性。[`04_Cursor操作.md`](./04_Cursor操作.md) を参照 |
| `@file` でファイル添付しても中身を読まない | Cursor のインデックス未完了。数十秒〜数分待ってから再試行 |

> 💡 Cursor 本体の起動・サインインのトラブルは **[`環境構築ガイド`](./02_環境構築ガイド.md)** 末尾のトラブルシューティングを参照してください。
