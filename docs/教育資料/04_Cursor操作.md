## はじめに：Cursor の画面（超基本）

エディタ（IDE）を初めて使う方向けに、画面の見かたと基本操作だけ先に押さえます。
**むずかしい機能は使いません。** 覚えるのは「① 左でファイルを開く ② 右の Chat に日本語で頼む」だけ。

```
   ┌──────────┬───────────────────────────┬──────────────┐
   │ 左：ファイル │ 中央：エディタ                │ 右：Chat (AI) │
   │ 一覧      │ （開いたファイルの中身を見る）    │ 日本語で頼む  │
   │          │                            │              │
   │ README   │  ここに本文が表示される         │ 「○○して」    │
   │ src/...  │                            │ と入力 → Enter │
   └──────────┴───────────────────────────┴──────────────┘
```

| やりたいこと | 操作 |
|---|---|
| ファイルを開く | 左の一覧でファイル名をクリック |
| AI に頼む（メインの使い方） | `Ctrl+L` で右に Chat を開く → 日本語で入力 → Enter |
| 開いている文章の一部だけ直す | 直したい所を選択 → `Ctrl+K` |
| 入力を保存する | `Ctrl+S` |

> 💡 マウスでクリック、キーボードで日本語入力 — **普段の PC 操作と同じ**です。
> プログラムを書く必要はありません。AI に「日本語でお願いする」だけ。

## 1. Cursor Hobby 枠

### 1.1 Hobby 枠とは（事実の正本）

Cursor の **無料プラン（Free）**。新規アカウントは自動的に Hobby 枠から始まり、月単位で **slow request** を一定数使える。

| 確認ポイント | Hobby 枠 |
|---|---|
| 料金 | 無料（Free） |
| クレジットカード登録 | **不要** |
| SMS（電話番号）認証 | **必要**（アカウント作成時の本人確認。課金とは無関係） |
| GitHub アカウント | **任意**（GitHub / Google / メールのいずれかでサインイン可。GitHub 必須ではない） |
| コードのデータ共有 | **必須ではない**（Privacy Mode を ON にすれば学習・保存に使われない） |
| リクエスト数 | 制限あり（下記 §1.2） |
| Tab 補完 | 制限あり（下記 §1.2） |

> 💡 サインイン手順とこの「安心ポイント」の解説は [`02_環境構築ガイド.md`](./02_環境構築ガイド.md) の §③。

### 1.2 上限値について

> ⚠️ Cursor の料金プラン・上限値（slow request 数・利用可能モデル等）は**頻繁に変動する**ため、本ファイルでは具体的な数値を固定しません。最新値は公式を確認してください。
> 公式: https://cursor.com/pricing ／ https://docs.cursor.com/

押さえておく性質だけ:

- Hobby は **エージェントのリクエスト数に制限あり**、**Tab 補完にも制限あり**。
- Fast request は基本的に有料プラン側の概念。Hobby は **slow request 中心**。
- Composer / Background Agent などは **Hobby では制限される**ことがある（§3 参照）。

### 1.3 Hobby 枠で快適に使うコツ

- **1 リクエスト 1 タスク** に絞る（複合タスクは小さく分割）
- **長文を 1 回で投げない**（プロンプトメモのセクション単位でコピペ）
- **多並列で Chat を立ち上げない**（同時実行は意味が薄く、上限を消費するだけ）
- `@file` / `@folder` を使って **AI に読ませる範囲を明示**（暗黙にリポジトリ全体を読まれると重い）
- 上限に近づいたら **待機**（月初にリセット）、または **Pro への移行** を検討（本キット外の判断）

## 2. Chat (Ctrl+L) — 基本

### 2.1 何に使うか

- 質問・調査
- 小さなコード片の生成
- 既存ファイルの説明・要約
- テキスト整形（メール文面、議事録、表形式変換 など）

### 2.2 使い方

1. `Ctrl+L` を押す（または右上の Chat アイコン）
2. 右ペインに Chat 欄が表示される
3. 入力欄にプロンプトを書いて Enter

### 2.3 ファイルを読ませる（`@` 記法）

Chat に頼む前に **どのファイル / フォルダを読ませるか** を `@` で指定できます。本講座で使うのはこの 2 つだけで十分です。

| 記法 | 効果 |
|---|---|
| `@file` | ファイル 1 つを選んで読ませる |
| `@folder` | フォルダごと読ませる（中の全ファイル） |

> 💡 `@` を打つとファイル候補が出るので、選ぶだけ。`@docs` `@web` `@git` など他の記法もありますが、**事務処理では使いません**（覚えなくて OK）。

`@file` の書き方の例（実際のタスク本文は [`src/office-task/README.md`](../../src/office-task/README.md) を参照）:

```
@file src/office-task/data/新規 データ1.csv の内容を読み、まずこのファイルが何のデータかを 1 行で要約し、続いて主力スキル別の人数と月単金の平均を Markdown 表で出してください。
```

```
@folder src/office-task/data の txt ファイル群（中身は契約書とスキルシート）を読み、それぞれの要点を 1 行で要約し、推奨ファイル名案も併せて提示してください。
```

## 3. 本講座では使わない機能（読み飛ばして OK）

Cursor には上級者向けの機能もありますが、**事務処理の体験チュートリアルでは一切使いません**。
画面に出てきても気にしなくて大丈夫です。

| 機能 | 何のためのもの | 本講座では |
|---|---|---|
| Composer (Ctrl+I) | 複数ファイルをまたぐ変更 | 使わない（Chat で十分） |
| Background Agent / クラウドエージェント | AI に裏で長時間作業させる | 使わない |
| MCP / スキル / フック | 外部ツール連携や自動化の拡張 | 使わない |

> 💡 覚えるのは **Chat（§2）** と **Inline Edit（§4）** の 2 つだけ。それ以外は不要です。

## 4. Inline Edit (Ctrl+K) — ピンポイント編集

### 4.1 何に使うか

- エディタで開いているファイルの **選択範囲だけ** AI に書き換えてもらう
- 「この行を Markdown 表にして」「この段落を敬語にして」など局所的な編集

### 4.2 使い方

1. 編集したい範囲を選択
2. `Ctrl+K`
3. 指示を入力 → Enter
4. 差分プレビューが出るので、`Accept` で反映、`Reject` で取り消し

## 5. Tab 補完

### 5.1 何に使うか

- 入力中の **次の数行**を AI が予測して提示
- そのまま採用するなら `Tab` キー
- 不要なら `Esc` または入力を続ける

### 5.2 Hobby 枠での扱い

Tab 補完は Hobby 枠でも基本的に使える（制限あり・公式 docs を確認）。事務処理タスクでは Markdown や CSV を編集中に **次のセル / 項目** を予測してくれるので有用。

## 6. Rules（`AGENTS.md` と `.cursor/rules/*.mdc`）

### 6.1 仕組み

Cursor は以下を **Chat / Composer のコンテキストに自動付与** する:

- リポジトリルートの `AGENTS.md`
- `.cursor/rules/*.mdc` 配下のルールファイル

本キットには両方とも同梱済み。ユーザーが明示的に `@file` で貼る必要はない。

### 6.2 本キットで適用されるルール（要約）

- **ランタイム非インストール原則**: Python / Node.js のインストールを伴う解を提案しない
- **Windows 標準機能優先**: PowerShell / Excel / エクスプローラ / Edge で完結する解を案内
- **元ファイル不変更**: ファイル操作タスクではコピー版を新名で保存
- **日本語回答**: ドキュメント・回答はすべて日本語

詳細は [`../../AGENTS.md`](../../AGENTS.md) と [`../../.cursor/rules/repo.mdc`](../../.cursor/rules/repo.mdc) を参照。

### 6.3 ルールが効いていないと感じたら

- Chat 上部に「Rules applied」「Applied rules」のような表示があるか確認
- 無ければ Open Folder を一度閉じて開き直す
- それでも効かないなら、Chat に明示的に `AGENTS.md のルールに従って再回答してください` と追記

## 7. 事務処理タスクでよく使う組み合わせ

| やりたいこと | 推奨 Cursor 機能 | 補助 |
|---|---|---|
| サンプル CSV を要約して欲しい | Chat + `@file` | — |
| データフォルダ全体の中身を一覧化 | Chat + `@folder` | — |
| Markdown 議事録のテンプレ作成 | Chat | — |
| 既存テキストを敬語に書き換え | Inline Edit (Ctrl+K) | — |
| ファイル整理のための PowerShell スクリプト生成 | Chat | 生成後にエディタで保存し、PowerShell で手元実行 |
| Excel 数式 / Power Query の手順書 | Chat | Excel 側で手順を再現 |

## 8. 統合トラブルシューティング

> 本キットの**全工程のトラブル対処をここに集約**しています。各手順ドキュメント（01〜03）からは本セクションへ誘導します。
> 該当フェーズの手順そのものは 01〜03 を参照。

### 8.A 導入・インストール

| 症状 | 対処 |
|---|---|
| `.exe` 実行で「Windows によって PC が保護されました」 | 「詳細情報」→「実行」。または `.exe` を右クリック → プロパティ →「ブロックの解除」にチェック → OK 後に再実行 |
| インストールが途中で失敗する | ダウンロード破損の可能性。`.exe` を削除し https://cursor.com/ から再取得 |

### 8.B サインイン（Hobby = 無料）

| 症状 | 対処 |
|---|---|
| サインインのブラウザが開かない | 既定ブラウザを Edge / Chrome に設定。または Cursor 画面の URL を手動でブラウザに貼る |
| サインイン後 Cursor 本体に戻らない | ブラウザの「Cursor を開きますか？」を許可。または Cursor を再起動 |
| SMS 認証コードが届かない | 電話番号と国コードを確認し、数分待って再送。アカウント作成には SMS 認証が必要（§1.1） |

### 8.C リポジトリ取得・Open Folder

| 症状 | 対処 |
|---|---|
| Clone repo で `Authentication failed` | パブリックリポジトリは認証不要。URL が `https://github.com/kurosawa-kuro/study-cursor.git` 形式か確認。プライベート配布なら HTTPS + Personal Access Token |
| `git: command not found`（PowerShell） | git 未導入。[git-scm.com](https://git-scm.com/) の Git for Windows を入れるか、Cursor の `Clone repo` を使う |
| Open Folder してもファイルツリーが空 | 親フォルダを開いている。展開後の `study-cursor/` フォルダを直接開く |
| 「このフォルダを信頼しますか？」を `No` にした | `File → Manage Workspace Trust → Trust this folder` で許可し直す |

### 8.D Chat・動作・Hobby 枠

| 症状 | 対処 |
|---|---|
| Chat (Ctrl+L) が開かない | キーバインド競合の可能性。右上の Chat アイコン、または `View → Chat` |
| 応答に Python コードや `pip install ...` が混ざる | `AGENTS.md` / `.cursor/rules` が未適用。Open Folder を開き直す。または「ランタイム非インストール原則に従い、Python を使わず PowerShell / Excel / Cursor 内処理で完結する解を再提示して」と追記 |
| `@file` で添付しても中身を読まない | Cursor のインデックス未完了。数十秒〜数分待って再試行 |
| `Too many slow requests` / 上限に到達 | 月初のリセットまで待機。緊急なら Pro 検討（本キット外） |
| 応答が極端に短い / 質が落ちる | フォールバックモデルに切り替わっている可能性。モデル選択 UI を確認 |
| Composer がプレミアム機能と表示される | Hobby 枠の対象外。Chat / Inline Edit で代替 |
