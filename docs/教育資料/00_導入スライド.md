## AI で事務処理を効率化する

### 〜 Cursor 入門講座 〜

```
   対象     : IT エンジニア派遣（SES）の営業担当者（非エンジニア OK）
   前提環境 : Windows 10 / 11（Mac でも読み替えで可）
   費用     : Cursor の無料枠（Hobby）だけで完結。クレジットカード不要
   登録     : SMS 認証のみ必要 / GitHub は不要（メールで OK）
   安全性   : データ共有は必須でない（Privacy Mode で学習オフ）
   所要時間 : 導入 15 分 → あとはプロンプトを貼るだけ
```

**この講座は「AI プログラミング講座」ではありません。**
メール作成・議事録・CSV 整理などの **定型事務を効率化する入口講座**です。

## 1. AI で業務効率化しよう

AI は「コードを書く道具」である前に、
**毎日の事務作業を肩代わりさせる道具**です。

```
   ❌ 思い込み : AI = エンジニアがコードを書くためのもの
   ⭕ 実際    : AI = 日本語で頼めば事務作業をこなしてくれる相棒
```

- お礼メールの文面を書いてもらう
- 商談メモから議事録テンプレを起こす
- CSV を読ませて集計表にする

**キーボードで日本語の指示を出すだけ。** これが本講座の出発点です。

---

## 2. なぜ "AI プログラミング" ではなく "業務効率化" か

プログラミングを覚える講座ではありません。
狙いは **「コードが書けなくても、日本語で指示すれば動く」**状態です。

| よくある誤解 | この講座のスタンス |
|---|---|
| プログラミングを学ばないと使えない | 学ばなくてよい。日本語の指示で十分 |
| 環境構築が大変そう | ランタイム不要。Cursor を入れるだけ |
| エンジニア向けのツール | 営業事務の定型作業のための道具として使う |

> 目的は「開発できるようになること」ではなく、
> **今日の事務作業を今日から軽くすること**。

---

## 3. 一般的な IT 業務（営業事務）の中身

SES 営業の事務作業は、その多くが **繰り返しの定型作業**です。

```
   📨 顧客対応        … お礼メール / 提案メール / 商談議事録
   👥 エンジニア・案件 … 一覧の集計レポート / 案件マッチング
   📑 提案資料        … スキルシート・契約書の要約・構造化
   💰 ベンダー比較     … 協力会社の見積を比較表にまとめる
   🗂 ファイル整理     … スキルシートのリネーム / 種類別フォルダ分け
```

**これらは「考える仕事」より「整える仕事」が中心。**
→ AI に肩代わりさせる余地が大きい領域です。

## 4. この講座で扱う業務課題

実際に体験する 7 つの事務タスク（すべて Cursor に日本語で依頼）:

1. 商談後の **お礼・提案メール** 文面生成
2. 商談 **議事録テンプレ** 生成
3. エンジニア一覧 / 案件一覧の **集計レポート**
4. エンジニア × 案件の **マッチング**（CSV 結合）
5. **スキルシート・契約書の要約**
6. 協力会社 **見積の比較表**化
7. **ファイル整理**（リネーム / フォルダ分け）

## 6. なぜ VS Code ではなく Cursor か

VS Code は素のエディタ。Cursor は **AI が組み込まれたエディタ**です。

| | VS Code | Cursor |
|---|---|---|
| AI チャット | 拡張頼み | 標準搭載（Ctrl+L） |
| ファイルを見ながら依頼 | 手動 | `@file` で直接読ませる |
| ルールの自動適用 | なし | `AGENTS.md` / `.cursor/rules` を自動付与 |

**Cursor は「Python を使わずに解いて」「日本語で答えて」といった制約を
ルールとして自動で効かせられる** → 非エンジニアが暴走しにくい。

---

## 7. なぜ Claude Code / Codex / Copilot ではなく Cursor か

同じ AI 支援でも、**入口の敷居**が違います。

```
   Claude Code / Codex (CLI 系)
     → 黒い画面（ターミナル）操作が前提。非エンジニアには敷居が高い

   GitHub Copilot
     → コード補完が主役。文章・データ整形の対話には向きにくい

   Cursor   ← 本講座が採用
     → 画面操作で完結（GUI）。無料の Hobby 枠で始められる
     → ファイルを見ながら、日本語で「やって」と頼める
```

> 非エンジニアが **ターミナル無し・無料で・日本語で** 始められること。
> これが Cursor を選ぶ理由です。

## 8. 基本は Windows 前提。ただし Mac でも実施できます

この教材は事務担当者の PC 環境を想定し、**Windows 10 / 11 を公式前提**にしています。

```
   Windows を前提にする理由
   ・事務部門では Windows 利用が多い
   ・Excel / エクスプローラ / PowerShell との相性がよい
   ・操作画面やファイルパスの説明を揃えやすい
```

**Mac でも Cursor は動くため、同じ考え方で進められます**（公式サポート外の補足）。
ただし以下は読み替えが必要です:

```
   エクスプローラ → Finder
   PowerShell    → ターミナル
   Windows パス   → Mac のパス表記
```

## 9. 実際のチュートリアルでやること

セットアップは **Cursor を入れる 1 回だけ**。あとはプロンプトを貼るだけです。

```
   【Phase 1】Cursor 導入（一度だけ・10〜20 分）
     ・cursor.com から .exe をダウンロードしてインストール
     ・サインイン（Hobby = 無料）
            │
            ▼
   【Phase 2】リポジトリを開いて使う（毎回・数分）
     ・Cursor で Open Folder
     ・Chat (Ctrl+L) を開く
     ・用意されたプロンプトを貼って送信
            │
            ▼
   🎉 メール文面・集計表などが返ってくる
```

→ 最短ルートは [`01_クイックスタート.md`](./01_クイックスタート.md)（15 分版）。

## 12. この講座で扱わないこと

期待値を揃えるため、**やらないこと**も明示します。

```
   ✗ プログラミングそのものの学習
   ✗ Python / Node.js などランタイムの導入
   ✗ Cursor 以外の AI ツール
   ✗ Windows 以外の OS への本格対応（Mac は読み替えの範囲）
   ✗ 社内システム連携（社内 API / SaaS 連携）
   ✗ 本格的な開発基盤（CI/CD・テスト・IaC など）
```

> あくまで **営業事務の定型作業を Cursor で軽くする入口**にフォーカスします。
