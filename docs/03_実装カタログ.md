## 2. `src/office-task/`（IT エンジニア派遣営業向け事務処理プロンプト集）

`README.md` に Cursor Chat / Composer へ貼り付けるためのプロンプト雛形を保管。フォーマットは `## プロンプトN: <概要>` + 箇条書き。難易度 Tier 順に並べてある。

| 番号 | 概要 | Tier | 対象データ | 主な実現手段 |
|---|---|---|---|---|
| プロンプト1 | 商談後のお礼・提案メール文面生成 | T1 | 入力情報のみ（任意で `新規 テキスト ドキュメント2.txt`（スキルシート）の中身を流用） | Cursor Chat 内で完結（テキスト生成のみ） |
| プロンプト2 | 商談議事録テンプレ生成（Markdown） | T1 | 入力情報（商談タイトル・参加者など） | Cursor Chat 内で完結。`data/` に `.md` として保存 |
| プロンプト3 | スキルシート・契約書の構造化・要約 | T2 | `data/新規 テキスト ドキュメント1.txt`（契約書）/ `data/新規 テキスト ドキュメント2.txt`（スキルシート） | Cursor Chat 内で完結。契約書 → 階層 Markdown、スキルシート → 提案用の項目別表 |
| プロンプト4 | エンジニア一覧 / 案件一覧の集計レポート | T2 | `data/新規 データ1.csv`（エンジニア一覧）+ `data/新規 データ2.csv`（案件一覧） | 小さいので Chat に `@file` で添付 → AI が直接集計し Markdown を出力 |
| プロンプト5 | ファイル名リネーム（曖昧名 → 内容に合った名前へ） | T3 | `data/` 4 ファイル全部（**曖昧名**） | Cursor が中身を読み、新ファイル名表 + PowerShell `Copy-Item` ワンライナーを提示。**元ファイル不変更**。このプロンプトのために 4 ファイルは意図的に曖昧名 |
| プロンプト6 | ファイル整理（拡張子別にコピー） | T3 | `data/` 4 ファイル | Cursor が PowerShell `Copy-Item` スクリプトを生成。`data/_整理/csv/` `data/_整理/txt/` に振り分け |
| プロンプト7 | エンジニア × 案件マッチング（CSV 結合） | T4 | `data/新規 データ1.csv`（エンジニア一覧）+ `data/新規 データ2.csv`（案件一覧） | Excel Power Query の手順、または `XLOOKUP` / `FILTER` / `LET` / `BYROW` 数式を Cursor が提示 |
| プロンプト8 | 協力会社見積書 PDF → 比較用 CSV（複数ベンダー） | T5 | `data/見積書/見積書_A社.pdf` / `見積書_B社.pdf` / `見積書_C社.pdf`（架空 SES 3 社同梱） | テキスト PDF: Edge コピペ／画像化 PDF: PowerToys Text Extractor／一括化: Word COM の PowerShell（任意・Office 必要）→ Cursor Chat で各 PDF を CSV 1 行に抽出 → 比較用 CSV にまとめる |

共通前提:

- **ランタイム非インストール原則**（`docs/01_仕様と設計.md §5.5`）に従い、Python / Node.js は使わない
- **Windows 標準機能**（PowerShell / Excel / エクスプローラ / Edge）と **Cursor 内処理** で完結する
- **元ファイルは絶対に変更・移動・削除しない**（コピー方式）
- 回答は日本語

`data/` のサンプル:

> ⚠️ ファイル名は **わざと Windows デフォルトの曖昧名** にしてある（プロンプト5 のリネーム課題が成立するため）。中身を「ファイル名」列ではなく「中身（実体）」列で把握する。

| ファイル名（曖昧・実際の名前） | 中身（実体） |
|---|---|
| `新規 データ1.csv` | 自社・協力会社の稼働中エンジニア 10 名のサンプル CSV（engineer_id / name / age / primary_skill / secondary_skills / years_total / monthly_rate / available_from / remote_pref / location / english_level / last_project / status ほか） |
| `新規 データ2.csv` | クライアントから来ている募集中案件 10 件のサンプル CSV（project_id / client / industry / position / must_skills / want_skills / monthly_rate_min / monthly_rate_max / start_date / duration_months / work_location / remote_type / openings / deadline / status） |
| `新規 テキスト ドキュメント1.txt` | 架空の SES 業務委託基本契約書サンプル（章立て構造: 総則 / 委託内容 / 対価 / 期間 / 秘密保持 / 知的財産 / 反社条項 / 契約終了 / 一般条項） |
| `新規 テキスト ドキュメント2.txt` | 架空エンジニア「山田太郎」（経験9年 Java バックエンド）のスキルシート（直近案件・技術スタック・希望単金・最短稼働日・資格 など） |
| `見積書/見積書_A社.pdf` | 架空の SES 見積書サンプル（株式会社さっぽろシステムズ、Java シニア・月単金 75 万円・6 ヶ月・週3リモート可、税込 5,038,000 円、有効期限 2026-06-30）。日本語フォント `IPAGothic` で生成 |
| `見積書/見積書_B社.pdf` | 架空の SES 見積書サンプル（合同会社ノーザン・ソフト、Java テックリード・月単金 82 万円・6 ヶ月・フルリモート・英語可、税込 5,412,000 円、有効期限 2026-07-15）。日本語フォント `IPAGothic` で生成 |
| `見積書/見積書_C社.pdf` | 架空の SES 見積書サンプル（フロンティアITソリューションズ株式会社、Java ミドル・月単金 68 万円・6 ヶ月・出社必須・引き継ぎドキュメント込み、税込 4,620,000 円、有効期限 2026-05-31）。日本語フォント `IPAGothic` で生成 |
