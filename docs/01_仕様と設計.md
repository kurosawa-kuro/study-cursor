# 01. 仕様と設計

本リポジトリ（**Cursor Hobby 枠 事務処理スターターキット**）の対象ユーザー・ゴール・全体構成。新規参加者および LLM が最初に読むことを想定。

---

## 1. 想定ユーザー

- **IT エンジニア派遣（SES）の営業担当者**（主想定）
- **Windows 10/11 専用**
- 基本的な PC 操作（ブラウザ・Excel・エクスプローラ）はできる
- プログラミング経験は不要
- **Cursor（Hobby 枠 = 無料プラン）** を営業事務に取り入れたい

---

## 2. ゴール

ユーザーが Windows PC 1 台で、**Cursor Hobby 枠だけ** を使って事務処理を自動化できる状態にする。

| 業務シーン | 自動化できる例 |
|---|---|
| 顧客対応 | 商談後のお礼・提案メール文面生成、商談議事録テンプレ生成 |
| エンジニア・案件管理 | エンジニア一覧 / 案件一覧の集計レポート、エンジニア × 案件マッチング（CSV 結合） |
| 提案資料作成 | スキルシート・契約書の構造化・要約 |
| ベンダー比較 | 協力会社見積書 PDF → 比較用 CSV（複数ベンダー） |
| ファイル整理 | ファイル名リネーム（スキルシート整理）、拡張子別フォルダにコピー |

「コードが書けなくても、日本語で Cursor Chat に指示すれば動く」を目指す。

---

## 3. スコープ

### 3.1 本リポジトリが扱う範囲

- **Cursor 導入ガイド**（Windows へ Cursor 本体をインストールしサインインするまでの手順書）
- **事務処理タスク用プロンプトメモ**（Cursor Chat / Composer にコピペして使う日本語プロンプト集）
- **プロンプトと一緒に使うサンプルデータ**（CSV・テキスト）
- **Cursor 用ルールファイル**（`.cursor/rules/*.mdc` と `AGENTS.md`）

### 3.2 本リポジトリが扱わない範囲

- **Python / Node.js などプログラミングランタイムのインストール**（本キットの基本フローでは使わない方針）
- プログラミング学習タスク
- Cursor 以外の AI ツール
- Windows 以外の OS
- Cursor 本体の実装
- 業務固有のシステム連携（社内 API、SaaS 連携など）
- 本格的なソフトウェア開発フレームワーク（CI/CD・テスト基盤・IaC など）

## 5. 設計上の決め事

### 5.1 元データは絶対に変更・移動・削除しない

`src/office-task/` の事務処理タスクで `data/` 配下を扱う場合、**元ファイルは触らず、コピーして新しい名前を付けて同ディレクトリに保存する** のが既定方針。Cursor Chat に依頼するときも、この前提を明示する。

### 5.2 プロンプトメモはセクションごとに自己完結させる

`src/office-task/README.md` のプロンプトメモは、**共通要件を上部に DRY 化せず**、各セクションで重複していても自己完結させる。利用者が単独でコピペして Cursor Chat に渡せるようにするため。

### 5.3 言語

ドキュメント・回答・コミットメッセージはすべて **日本語**。読者は日本語話者の非エンジニアを想定。

### 5.4 Cursor Hobby 枠の運用前提

本キットは Cursor の Hobby 枠（無料プラン）で完結することを前提に設計する。

- Slow request の月次上限・利用可能モデルの制限を意識する（具体的な数値は [`./教育資料/04_Cursor操作.md`](./教育資料/04_Cursor操作.md) 参照）
- 1 リクエストに長文や多並列を詰め込まず、小さく分割して依頼する
- 上限に近づいたら待機 or Pro への移行を検討（本キット外）

### 5.5 ランタイム非インストール原則

**Python / Node.js / Docker などのプログラミングランタイムは原則インストールしない**。Cursor Chat / Composer に対しては「ランタイム導入を伴う解は提案しない」「Windows 標準機能（PowerShell / Excel / エクスプローラ / Edge）または Cursor 内のファイル読み書きで完結する解を案内する」前提で依頼する。

これにより、非エンジニアでも環境構築の障壁なしに事務処理自動化を始められる。

### 5.6 Cursor を主に動かす

実装作業は Cursor の Chat (Ctrl+L) / Composer (Ctrl+I) / Inline Edit (Ctrl+K) / Tab 補完に集約する。CLI ベースの AI ツールは使わない。
